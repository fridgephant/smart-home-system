# 论文与工程代码对照说明

## 论文设计描述
- 核心控制器：ESP32-WROOM-32E
- 传感器：SHT30 温湿度传感器、BH1750 光照传感器、SGP30 空气质量传感器
- 显示：0.96 英寸 OLED 屏幕
- 通信：ESP32 Wi-Fi + MQTT 协议上传云服务器
- 控制逻辑：温度超阈值启风扇、光照不足启补光灯、空气质量超标启净化器
- 远程控制：手机 APP 远程查看数据、设置阈值、手动控制继电器
- 软件框架：Arduino
- 测试方法：Monte Carlo 随机测试控制逻辑触发概率

## 当前工程实现
- `src/main.cpp`
  - 使用 ESP32 Arduino 开发，包含 WiFi、MQTT、OLED 显示、SHT30、BH1750、SGP30
  - 实现了环境数据采集、OLED 显示、MQTT 上传、自动继电器控制
  - 支持远程 MQTT 命令：阈值设置和手动开关控制
- `inc/main.h`
  - 声明了 Arduino/ESP32 项目所需函数接口
- `tests/monte_carlo_test.py`
  - 增加了与论文逻辑一致的 Monte Carlo 测试脚本
  - 输出风扇、补光灯、净化器触发概率
- `Makefile`
  - 改为 Arduino CLI 编译方式，适配 ESP32 开发环境
- `README.md`
  - 更新为论文风格项目说明，包含开发环境、编译依赖、测试方案

## 关键一致性验证
- 传感器类型和显示模块与论文一致
- MQTT 上传和远程阈值控制逻辑与论文一致
- 自动联动控制规则与论文一致：
  - `temperature > thresholdTemp` 启风扇
  - `luxValue < thresholdLight` 启补光灯
  - `tvoc > thresholdTVOC || co2 > thresholdCO2` 启净化器
- Monte Carlo 测试脚本采用随机环境样本统计触发概率，符合论文描述
